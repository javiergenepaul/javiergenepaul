# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

This is a **GitHub special profile repository** (`javiergenepaul/javiergenepaul`). Because the repo name matches the username, GitHub renders `README.md` on the owner's profile page. There is no application, build system, package manager, or test suite — the deliverable is the rendered `README.md`.

## Structure

- `README.md` — the entire profile. Authored as a playful pseudo–React app: each section is a Markdown heading styled as a fake source file (e.g. `src/pages/AboutMe.tsx`, `src/pages/TechStack.tsx`). These paths are decorative; no such files exist.
- `.github/workflows/snake.yml` — scheduled GitHub Action.

## README conventions to preserve

- The top banner is ASCII art generated by a [figlet-style](https://patorjk.com) generator. When editing it, keep all six rows of each glyph row aligned — these are box-drawing characters, so column alignment is fragile and a single off-by-one breaks the whole figure.
- Section headers follow the `### <emoji> \`src/pages/<Name>.tsx\` <emoji>` pattern. Match it when adding sections.
- The tech-stack grid uses `<table>` with `skillicons.dev` / `go-skill-icons.vercel.app` icon URLs (`?i=<slug>`). Add a tech by copying a `<td align="center" width="96">` cell and swapping the slug.
- Stats/Pac-Man images are live external services (`github-profile-summary-cards.vercel.app`, `komarev.com` view counter). They render only on GitHub, not in most local Markdown previews — don't assume a broken local preview means broken Markdown.

## The Pac-Man animation (`.github/workflows/pacman.yml`)

Runs daily (`cron: "0 0 * * *"`), on `push` to `main`, and on manual `workflow_dispatch`. It uses `abozanona/pacman-contribution-graph` to generate a contribution-graph "Pac-Man" SVG into `dist/`, then `crazy-max/ghaction-github-pages` commits the output to a dedicated `output` branch. The README references the result at:

```
https://raw.githubusercontent.com/javiergenepaul/javiergenepaul/output/pacman-contribution-graph-dark.svg
```

Key implications:
- The `output` branch is **build output**, not source — do not hand-edit it; the action overwrites it.
- The job needs `permissions: contents: write` to push the `output` branch — keep it.
- The action supports other `games:` (`breakout`, `galaga`, `puzzle-bobble`, `bomberman`, `minesweeper`); each emits `dist/<game>-contribution-graph[-dark].svg`. Match the README image filename to the game.
- If the image breaks, check the Action run and that the `output` branch exists, rather than editing the README link.
- To preview changes without waiting for the daily schedule, trigger the workflow manually from the Actions tab (`workflow_dispatch`).
